Insurance-Fraud-Detection
End-to-end Insurance Fraud Detection project using Python, Machine Learning, and Power BI. Performed data cleaning, EDA, feature engineering, fraud prediction using a Decision Tree model, and built an interactive dashboard with actionable business insights for insurance claim fraud analysis.

This is an end-to-end Insurance Fraud Detection project. The main goal is to identify whether an insurance claim is fraudulent or not using Machine Learning and to create an interactive Power BI dashboard for business insights.

Tools Used

- Python
- Pandas
- NumPy
- Matplotlib
- Scikit-learn
- Power BI

Steps

- Data Cleaning
- Exploratory Data Analysis (EDA)
- Feature Engineering
- Decision Tree Model
- Model Evaluation
- Power BI Dashboard

Dashboard Highlights

- Total Customers
- Total Claims
- Fraud Cases
- Model Accuracy
- Age Distribution
- Fraud by Incident Type
- Claim Amount by Severity
- Top 5 Vehicle Brands
- Fraud Cases by State

Model Used

- Decision Tree Classifier
- Accuracy: **77.5%**

Key Insights

- Most customers belong to the 30–45 age group.
- Multi-vehicle and single-vehicle collisions have more fraud cases.
- Major damage claims have the highest claim amount.
- Fraud cases are concentrated in a few states.
